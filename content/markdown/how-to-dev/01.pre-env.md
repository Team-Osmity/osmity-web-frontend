# 前提環境
Kairo.tsでは、主に以下のツールを使用します。
- Git
- Node.js
- (VSCode)

これらの環境を用意することが、Kairo.tsを扱う上での最低条件となります。
※基本的にPCでの環境整備が前提となります。

### Git のインストール
とりあえず、開発協力者がみんなWindows環境っぽいので、windowsだけで説明します。

Gitとは少し詳しく言うと、**ローカル用のバージョン管理ツール**です。
GitとGitHubは別物なので注意してください。
※GitHubはGitをweb上でみんなで使えるようにしたものって感じです。

以下のリンクにアクセスし、**Git for Windows/x64 Setup** をクリックして、
Gitのインストーラーをダウンロードしてください。
https://git-scm.com/install/windows

ダウンロードしたインストーラーのアプリを起動します。
起動後に、利用規約や設定などをたくさん聞かれますが、全部デフォルト設定で良いので、
全部「Next」連打で大丈夫です。

### Node.js のインストール
Node.jsは **javascriptの実行環境**です。
開発時に使用するデプロイのスクリプトを実行したり、また Typescript での開発のために使用します。

以下のリンクにアクセスして、**Windowsインストーラー(.msi) をダウンロードしましょう。
https://nodejs.org/ja/download

ダウンロードしたインストーラーのアプリを起動して、
Git同様に全てデフォルト設定で大丈夫なので、「Next」連打してください。

途中で3つの項目が現れて、「chagne」「repair」「remove」を選ばれますが、
「change」を選択しましょう。

### VSCode のインストール
VSCodeは、**テキストエディタ**です。
もし、別で使っているテキストエディタがある場合はそちらでも構いません。
この記事では、そもそもテキストエディタも使ったことが無い初心者の方のために、
今現在静空が使用しているエディタを紹介しています。

以下のリンクにアクセスして、**Download for Windows**
を選択しましょう。

(VSCodeのセットアップは忘れたので、詳しく言えないですが、
とりあえずGit, Node.js同様にインストーラーでデフォルト設定を選択してやればいいと思います)

## 確認
Windowsで Powershell を起動して、以下のコマンドをそれぞれ実行してみてください
`git -v`
`node -v`
`npm -v`
もし、npm -vが実行できない場合は、以下のコマンドを実行したのちに、もう一度実行してみてください
`Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned`

それぞれ、バージョンが表示されると成功です！
